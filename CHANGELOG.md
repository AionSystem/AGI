## Versioning Convention

This repository uses **convergence-state driven versioning**:

| Version Range | Convergence State | Description |
|--------------|-------------------|-------------|
| 0.x.x | M-NASCENT | Specification phase. Unmapped zones predominate. |
| 1.x.x | M-MODERATE | Validation active. First FCL entries confirmed. |
| 2.x.x | M-STRONG | Validated. Deployment confidence justified. |

Major version increments require convergence state advancement. Minor versions add documented rooms or specifications. Patch versions correct errors or clarify without structural change.
